# TreeEdit նկարագրություն

## Ներածություն

Համակարգում ծառ-տեղեկատու ավելացնելու և ցուցադրելու համար բավական է Աշտ-ի նկարագրման json-ում ավելացնել **TreeBrowser** տեսակի (type) հանգույց։ Հանգույցի name դաշտում անհրաժեշտ է նշել ծառի ներքին անունը (կոդը)։

**Օրինակ**

```json
"nodes": [
  {
    "armenianCaption": "Մասնաճյուղեր",
    "englishCaption": "Filials",
    "name": "ACSBRNCH",
    "type": "TreeBrowser"
  }
]
```

Եթե ծառի համար **TreeEdit** դաս սահմանված չէ, ապա UI-ում ծառ-տեղեկատուն բացելիս համակարգը լռությամբ կիրառում է հետևյալ վարքագիծը․
* Ծառ-տեղեկատուում ցուցադրվում են **name** դաշտում նշված ծառի հանգույցները։
* Ծառ-տեղեկատուի համակարգային կոնտեքստային ֆունկցիաների (**Ավելացնել**, **Դիտել/Խմբագրել**, **Հեռացնել**, ...) հասանելիությունը սահմանվում է ծառի նկարագրության (**Definition**) **AllowAddNode**, **AllowEdit**, **AllowDelete** և համանման հատկությունների միջոցով։

**TreeEdit** դաս անհրաժեշտ է սահմանել միայն այն դեպքում, երբ պահանջվում է իրականացնել լրացուցիչ կամ ոչ ստանդարտ տրամաբանություն։ Օրինակ եթե անհրաժեշտ է՝
* Ծառ-տեղեկատուում ցուցադրել custom տվյալներ (օրինակ՝ ծառի հանգույցների միայն մի մասը՝ որոշակի պայմանի հիման վրա)։
* Փոփոխել համակարգային կոնտեքստային ֆունկցիաների (**Ավելացնել**, **Դիտել/Խմբագրել**, **Հեռացնել**, ․․․) հասանելիությունը կամ վարքագիծը,
* Ծառում հանգույց ավելացնելիս կամ խմբագրելիս սահմանել լրացուցիչ վալիդացիա։

Նշված դեպքերում անհրաժեշտ է սահմանել **TreeEdit** դաս, որը պետք է ունենա **TreeEdit** ատրիբուտը՝ ծառի ներքին անունով (կոդով), և ժառանգի **TreeEdit** դասը։

**Օրինակ**

```c#
[TreeEdit("ApTrans")]
public class ApTransTreeEdit : TreeEdit
```
