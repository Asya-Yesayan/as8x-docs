# Առաջադրանքների փաստաթղթեր և դիտելու ձևեր

Առաջադրանքների սահմանման և կատարման գործընթացը իրականացվում է «Ադմինիստրատորի ԱՇՏ»-ի **«Առաջադրանքներ»** թղթապանակից։ Թղթապանակը նախատեսված է առաջադրանքների ձևանմուշների և ժամանակացույցերի սահմանման, ինչպես նաև դրանց կատարման ընթացքի և պատմության դիտման համար։

Առաջադրանքների կարգավորումները սահմանվում են [appsettings.json](../Project/appsettings_json.md) կոնֆիգուրացիոն ֆայլի [BackgroundJobs](../Project/appsettings_json.md#backgroundjobs) բաժնում, որի **Enabled** կարգավորումը ակտիվացնում է առաջադրանքների հերթագրման և կատարման մեխանիզմը։

## Առաջադրանքների ձևանմուշներ

Այս դիտելու ձևը գտնվում է «Ադմինիստրատորի ԱՇՏ» -> «Առաջադրանքներ» թղթապանակում: Նախատեսված է [առաջադրանքի ձևանմուշների](#առաջադրանքի-ձևանմուշ) ստեղծման, խմբագրման, հեռացման և [հերթագրման](#հերթագրել-կատարման) համար։

![job_templates_view](images/jobs/job_templates_view.png)

### Առաջադրանքի ձևանմուշ

Այս փաստաթուղթը ստեղծվում է [«Առաջադրանքների ձևանմուշներ»](#առաջադրանքների-ձևանմուշներ) դիտելու ձևի «Ավելացնել» կոնտեքստային ֆունկցիայի միջոցով։

| Ժամանակացույց          | Առաջադրանքի կատարման ժամանակացույցի կոդը։ Ակտիվանում է **«1 - Ժամանակացույցով»** գործարկման տեսակի դեպքում։ <br> Առաջադրանքի կատարման ժամանակացույցները սահմանվում են «Ադմինիստրատորի ԱՇՏ» -> [**«Առաջադրանքների ժամանակացույցներ»**](#առաջադրանքի-ժամանակացույց) դիտելու ձևի **«Ավելացնել»** կամ **«Պատճենել»** կոնտեքստային ֆունկցիայով:              |

| Նախորդ իրադարձություն  | Այն իրադարձության կոդը, որի առաջանալուց հետո գործարկվելու է ընթացիկ առաջադրանքը։ Ակտիվանում է **«3 - Իրադարձությունից հետո»** գործարկման տեսակի դեպքում։                          |
| Նշում                  | Նշումներ առաջադրանքի մասին։ Հնարավոր է ընտրել մեկից ավելի արժեքներ։ Նշումները սահմանվում են «Ադմինիստրատորի ԱՇՏ» -> «Առաջադրանքներ» -> «Նշումներ» -> **«Նշումներ առաջադրանքների համար»** դիտելու ձևի **«Ավելացնել»** կոնտեքստային ֆունկցիայով։            |

### Առաջադրանքի քայլեր

«Առաջադրանքի քայլեր» աղյուսակում սահմանվում են առաջադրանքում գործարկվող գործողությունները (տվյալների մշակման հարցում - DPR): 

| Անվանում               | Նկարագրություն                                                       |
| ---------------------- | -------------------------------------------------------------------- |
| Քայլ N                 | Գործողության համարը, որը սահմանում է դրա կատարման հաջորդականությունը։                                   |
| Ընդհատել սխալի դեպքում | Գործողությունում սխալների առաջացման դեպքում առաջադրանքի դադարեցման հայտանիշ։ |
| Գործողություն          | Գործողությունում կանչվող տվյալների մշակման հարցման (DPR) կոդը:                        |
| Գործողության անվանում  | Կանչվող տվյալների մշակման հարցման (DPR) հայերեն անվանումը: Լրացվում է ավտոմատ կերպով՝ «Գործողություն» դաշտը լրացնելիս և ենթակա չէ խմբագրման։          |
| Նշում                  | Նշումներ առաջադրանքի քայլի մասին։ Նշումները սահմանվում են «Ադմինիստրատորի ԱՇՏ» -> «Առաջադրանքներ» -> «Նշումներ» -> **«Նշումներ առաջադրանքների քայլերի համար»** դիտելու ձևի **«Ավելացնել»** կոնտեքստային ֆունկցիայով։                |

### Առաջադրանքի ընթացիկ քայլի պարամետրեր

«Առաջադրանքի ընթացիկ քայլի պարամետրեր» աղյուսակում սահմանվում են «Առաջադրանքի քայլեր» աելացված յուր․ գործողության (տվյալների մշակման հարցում - DPR) կատարման պարամետրերը, ծանուցման կարգավորումները։ 

Կատարման պարամետրերը լրացնելու համար անհրաժեշտ է սեղմել **«Խմբագրել»** կոճակը։

| Անվանում               | Նկարագրություն                                                       |
| ---------------------- | -------------------------------------------------------------------- |
| Պարամետրեր             | Կանչվող տվյալների մշակման հարցման (DPR) պարամետրերը։                 |
| Ծանուցման տարբերակ     | Քայլի ծանուցման տարբերակը։ <br> **0** - Երբեք։ <br> **1** - Միայն սխալների դեպքում։ <br> **2** - Միշտ։ <br> **2** արժեքի դեպքում ծանուցումներ են ուղարկվում քայլի կատարման սկզբի, բարեհաջող ավարտի, սխալների առաջացման և ձախողման դեպքերում, իսկ **1** արժեքի դեպքում` միայն սխալների առաջացման և ձախողման դեպքերում։ <br> Եթե և Առաջադրանքի ձևանմուշում, և քայլերի «Ծանուցման տարբերակ» դաշտում լրացված է **«Միայն սխալների դեպքում»** կամ **«Միշտ»** արժեքը, ապա ծանուցումը կկատարվի յուրաքանչյուրի համար առանձին–առանձին։      |
| Ծանուցվող խմբեր     | Ծանուցվող խմբերի կոդերի ցուցակը։ Ծանուցվող խմբերը սահմանվում են «Ադմինիստրատորի ԱՇՏ» -> «Համակարգային աշխատանքներ» -> [**«Ծանուցումների խմբեր»**](#ծանուցման-խումբ) դիտելու ձևի **«Ավելացնել»** կոնտեքստային ֆունկցիայով:     |

### Հերթագրել կատարման

Այս դիալոգը բացվում է [«Առաջադրանքների ձևանմուշներ»](#առաջադրանքների-ձևանմուշներ) դիտելու ձևի **«Հերթագրել կատարման»** կոնտեքստային ֆունկցիայի միջոցով։

![schedule_execution](images/jobs/schedule_execution.png)

Դիալոգի դաշտերը լրացվում են [«Առաջադրանքի ձևանմուշից»](#առաջադրանքի-ձևանմուշ)։ Հնարավոր է [«Առաջադրանքի քայլերից»](#առաջադրանքի-քայլեր) ընտրել, թե որոնք կատարվեն, և խմբագրել պարամետրերի արժեքները։

## Առաջադրանքի ժամանակացույց

Այս փաստաթուղթը նախատեսված է առաջադրանքների կատարման ժամանակացույցերի սահմանման համար։ Ավելացվում է **«Ադմինիստրատորի ԱՇՏ» -> «Առաջադրանքներ» -> «Առաջադրանքների ժամանակացույցեր»** դիտելու ձևի **«Ավելացնել»** կոնտեքստային ֆունկցիայով։ Սահմանված ժամանակացույցերը հետագայում օգտագործվում են [առաջադրանքի ձևանմուշներում](#առաջադրանքի-ձևանմուշ)՝ որպես **«Ժամանակացույց»** դաշտի արժեք։

Ժամանակացույցի դաշտերի համատեղման հատուկ դեպքերը՝

- Եթե «Սկզբի ամսաթիվ» և «Վերջի ամսաթիվ» դաշտերը լրացված չեն, սակայն «Կրկնել ամեն (օր)» դաշտում նշված է **1** արժեք, ապա առաջադրանքը կաշխատի անվերջ՝ «Կրկնել ամեն (օր)» դաշտում լրացված պարբերականությամբ։ **1**-ից մեծ արժեքի դեպքում «Սկզբի ամսաթիվը» պարտադիր է, քանի որ կրկնման հաշվարկը կատարվում է դրանից։
- Եթե «Կրկնել ամեն (օր)» դաշտում արժեք լրացված չէ, ապա առաջադրանքի կատարման հաճախականությունը որոշվում է մյուս ֆիլտրներով («Շաբաթական», «Ամսական» էջեր, «Սկզբի/Վերջի ամսաթիվ»)։ Եթե այլ ֆիլտր չկա, առաջադրանքը կաշխատի ամեն օր։
- Եթե նշված է միայն «Կատարման սկիզբը» առանց «Կրկնել (րոպե)» դաշտի, ապա առաջադրանքը կաշխատի միայն կատարման սկզբին, իսկ «Կատարման ավարտը» անտեսվում է։

| Ամսվա ֆիքսված օրեր | Ամսվա այն օրերը, որոնց առաջադրանքը կատարվելու է։ Արժեքները լրացվում են ստորակետով տարանջատված (օրինակ՝ 1,15,28)։ Կողքի **«Բացառել»** նշիչը նշված լինելու դեպքում առաջադրանքը կկատարվի ընտրված ամիսների բոլոր օրերին՝ բացառությամբ նշված օրերի։ |
| Ամսվա աշխատանքային օրեր | Ամսվա այն աշխատանքային օրերը (հաշվարկն արվում է աշխատանքային օրացույցով), որոնց առաջադրանքը կատարվելու է։ Արժեքները լրացվում են ստորակետով տարանջատված։ Կողքի **«Բացառել»** նշիչը նշված լինելու դեպքում առաջադրանքը կկատարվի ընտրված ամիսների բոլոր օրերին՝ բացառությամբ նշված օրերի։ |

## Հերթագրված առաջադրանքներ

Այս դիտելու ձևը գտնվում է «Ադմինիստրատորի ԱՇՏ» -> «Առաջադրանքներ» թղթապանակում: Դիտելու ձևում ցուցադրվում են արդեն հերթագրված և կատարման ընթացքում գտնվող առաջադրանքները՝ [«Սպասվող առաջադրանքներից»](#սպասվող-առաջադրանքներ) ի տարբերություն, որտեղ դեռևս չհերթագրված առաջադրանքներն են։ Նախատեսված է առաջադրանքների [ընդհատման](#հեռացնել-հերթագրված-առաջադրանքը), [կատարման մանրամասների](#կատարման-մանրամասներ), կատարման [քայլերի](#դիտել-ըստ-քայլերի) և [կատարման ընթացքի](#կատարման-ընթացք) դիտման համար։

![scheduled_jobs](images/jobs/scheduled_jobs.png)

### Հեռացնել հերթագրված առաջադրանքը

«Հերթագրված առաջադրանքներ» -> «Հեռացնել հերթագրված առաջադրանքը» կոնտեքստային ֆունկցիան կարելի է կանչել միայն այն առաջադրանքների համար, որոնք դեռ չեն գտնվում կատարման ընթացքում (միայն հերթագրված վիճակում գտնվող առաջադրանքների համար):

### Կատարման մանրամասներ

[«Հերթագրված առաջադրանքներ»](#հերթագրված-առաջադրանքներ)/[«Առաջադրանքների պատմություն»](#առաջադրանքների-պատմություն) -> «Կատարման մանրամասներ» կոնտեքստային ֆունկցիայի կանչի արդյունքում բացվում է տեքստային հաշվետվություն, որտեղ ցուցադրվում են առաջադրանքի (սերվիս, կատարող, վիճակ) և կատարման քայլերի մանրամասները (կատարման սկիզբ, վերջ, տևողություն, ...)։ 

Առաջադրանքի ամբողջական նկարագրությունը ներկայացված է [Առաջադրանքի ձևանմուշ](#առաջադրանքի-ձևանմուշ) բաժնում, իսկ կատարման քայլերինը՝ [Առաջադրանքի կատարման քայլեր](#առաջադրանքի-ընթացիկ-քայլի-պարամետրեր) բաժնում։

![job_execution_details](images/jobs/job_execution_details.png)

### Դիտել ըստ քայլերի

[«Հերթագրված առաջադրանքներ»](#հերթագրված-առաջադրանքներ)/[«Առաջադրանքների պատմություն»](#առաջադրանքների-պատմություն) -> «Դիտել ըստ քայլերի» կոնտեքստային ֆունկցիայի կանչի արդյունքում բացվում է «Ընթացիկ առաջադրանքներ (ըստ քայլերի)» դիտելու ձևը, որտեղ ցուցադրվում են առաջադրանքի կատարման քայլերի մանրամասները (կատարման սկիզբ, վերջ, տևողություն, ...)։ 

Կատարման քայլերի ամբողջական նկարագրությունը ներկայացված է [Առաջադրանքի կատարման քայլեր](#առաջադրանքի-ընթացիկ-քայլի-պարամետրեր) բաժնում։

![view_by_steps](images/jobs/view_by_steps.png)

### Կատարման ընթացք

[«Հերթագրված առաջադրանքներ»](#հերթագրված-առաջադրանքներ) -> «Կատարման ընթացք» կոնտեքստային ֆունկցիան հասանելի է այն առաջադրանքների համար, որոնք գտնվում են ոչ «Հերթագրված» վիճակում (այսինքն՝ կատարման ընթացքում, ինչպես նաև ավարտված/ընդհատված)։ Կանչի արդյունքում բացվում է առաջադրանքի կողմից գործարկվող DPR-ի կատարման պրոգրեսի պատուհանը, ինչպես նաև Status bar-ում ցուցադրվում է status text-ը։

Կատարման պրոգրեսի պատուհանի «Ընդհատել» կոճակի միջոցով հնարավոր է ընդհատել առաջադրանքի կատարումը։

## Սպասվող առաջադրանքներ

Այս դիտելու ձևը գտնվում է «Ադմինիստրատորի ԱՇՏ» -> «Առաջադրանքներ» թղթապանակում: Դիտելու ձևում ցուցադրվում են **«Ժամանակացույցով»** կամ **«Առաջադրանքից հետո»** գործարկման տեսակով այն առաջադրանքները, որոնք դեռևս չեն հերթագրվել կատարման (ի տարբերություն [«Հերթագրված առաջադրանքների»](#հերթագրված-առաջադրանքներ), որտեղ արդեն հերթագրված կամ կատարման ընթացքում գտնվող առաջադրանքներն են)։

**«Առաջադրանքից հետո»** գործարկման տեսակով առաջադրանքների համար «Կատարվելիք ժամանակ» դաշտը չի լրացվում, քանի որ այն կախված է նախորդ առաջադրանքի ավարտից։

![expected_jobs](images/jobs/expected_jobs.png)

## Առաջադրանքների պատմություն

Այս դիտելու ձևը գտնվում է «Ադմինիստրատորի ԱՇՏ» -> «Առաջադրանքներ» թղթապանակում: Դիտելու ձևում ցուցադրվում են բոլոր արդեն գործարկված առաջադրանքները (կատարման ընթացքում, բարեհաջող ավարտված, սխալով ավարտված, ընդհատված)։ Նախատեսված է առաջադրանքների [կատարման մանրամասների](#կատարման-մանրամասներ), կատարման [քայլերի](#դիտել-ըստ-քայլերի), [կատարման ընթացքում առաջացած սխալների](#կատարման-ընթացքում-առաջացած-սխալներ), [կատարման ընթացքում առաջացած սխալներ Seq-ում](#կատարման-ընթացքում-առաջացած-սխալներ-seq-ում) դիտման համար։

![job_history](images/jobs/job_history.png)

### Կատարման ընթացքում առաջացած սխալներ

[«Առաջադրանքների պատմություն»](#առաջադրանքների-պատմություն)/«Առաջադրանքների պատմություն (ըստ քայլերի)» -> «Կատարման ընթացքում առաջացած սխալներ» կոնտեքստային ֆունկցիայի կանչի արդյունքում բացվում է **«Առաջադրանքի կատարման մանրամասներ»** դիտելու ձևը, որտեղ ցուցադրվում են առաջադրանքի կատարման ընթացքում գրանցված բոլոր լոգերը։

Նշված դիտելու ձևից հասանելի են [«Արտահանել»](#արտահանել), [«Դիտել/խմբագրել»](#դիտելխմբագրել) կոնտեքստային ֆունկցիաները։

## Դիտել/խմբագրել

«Առաջադրանքի կատարման մանրամասներ» -> «Դիտել/խմբագրել» կոնտեքստային ֆունկցիան ցուցադրվում է առաջադրանքի կատարման ընթացքում գրանցված լոգի ամբողջական նկարագրությունը։

![job_error_log_function](images/jobs/job_error_log_function.png)

![job_error_log](images/jobs/job_error_log.png)

| Անվանում | Նկարագրություն |

| Լոգավորման մակարդակ | Գրանցված հաղորդագրության լոգավորման մակարդակը (Error, Warning, Info)։ |
| Հաղորդագրություն | Լոգում գրանցված հաղորդագրությունը, որը նկարագրում է տեղի ունեցած իրադարձությունը։ |
| Ամսաթիվ | Լոգի գրանցման ամսաթիվը և ժամանակը։ |
| Ծրագիր | Այն ծրագրի անունը, որի կողմից ստեղծվել է տվյալ լոգի գրառումը։ |

| API client | Այն կլիենտ ծրագրի ներքին նույնականացման համարը (id), որի շրջանակներում գրանցվել է լոգը։ |

## Դիտել պարամետրերը

«Առաջադրանքների պատմություն (ըստ քայլերի)» -> «Դիտել պարամետրերը» կոնտեքստային ֆունկցիայի կանչի արդյունքում բացվում է դիալոգ՝ readonly ռեժիմով, որտեղ ցուցադրվում են առաջադրանքի կատարման պարամետրերը։

![view_job_parameters](images/jobs/view_job_parameters.png)

![view_job_parameters_dialog](images/jobs/view_job_parameters_dialog.png)

### Կատարման ընթացքում առաջացած սխալներ Seq-ում

[«Առաջադրանքների պատմություն»](#առաջադրանքների-պատմություն)/«Առաջադրանքների պատմություն (ըստ քայլերի)» -> «Կատարման ընթացքում առաջացած սխալներ Seq-ում» կոնտեքստային ֆունկցիայի կանչի արդյունքում բացվում է կատարման սխալների լոգերը պարունակող Seq սերվերը։ Seq սերվերի հասցեն նշանակվում է SEQREADURL համակարգային պարամետրի միջոցով։         

## 8X վեբ ծառայություններ

Այս դիտելու ձևը գտնվում է «Ադմինիստրատորի ԱՇՏ» -> «Համակարգային աշխատանքներ» թղթապանակում: Նախատեսված է վեբ ծառայությունների (սերվիսների) ստեղծման, խմբագրման, հեռացման և ծառայության ասինխրոն պրոցեսների դիտման համար։

![web_services](images/jobs/web_services.png)

Վեբ ծառայությունները կիրառվում են [«Առաջադրանքի ձևանմուշներում»](#առաջադրանքի-ձևանմուշ)՝ նշելու համար այն սերվիսը, որի վրա պետք է գործարկվի տվյալ առաջադրանքը և «Ադմինիստրատորի ԱՇՏ» -> «Համակարգային աշխատանքներ» -> «Սերվիսային ասինխրոն պրոցեսներ»-ում՝ պրոցեսները ըստ սերվիսների տարանջատելու նպատակով։

![web_service](images/jobs/web_service.png)

| Անվանում | Նկարագրություն |
|---------|----------------|
| ID | Վեբ ծառայության (սերվիսի) նույնականացուցիչը (id): |
| Հասցե | Վեբ ծառայության (սերվիսի) հասցեն։ |
| Ներքին հասցե | Վեբ ծառայության (սերվիսի) լոկալ հասցեն։ |
| Դեր | Վեբ ծառայության (սերվիսի) դերը։ Վեբ ծառայությունը (սերվիսի) միաժամանակ կարող է ունենալ մեկ և ավելի դերեր։  <br> **Desktop** - Սերվիս, որը սպասարկում է desktop-ից եկող հարցումները: <br> **Integration** - Սերվիս, որը կապ է ապահովում արտաքին համակարգերի կամ այլ սերվիսների հետ: <br> **JobExecuter** - Սերվիս, որը նախատեսված է առաջադրանքների, ֆոնային առաջադրանքների կատարման համար։ <br> **Worker** - Ֆոնային պրոցես, որը կատարում է ծանր կամ երկարաժամկետ հաշվարկներ կամ հաղորդագրությունների մշակում։  |

## Ծանուցման խումբ

Ծանուցման խումբը ստեղծվում է «Ադմինիստրատորի ԱՇՏ» -> «Համակարգային աշխատանքներ» -> **«Ծանուցումների խմբեր»** դիտելու ձևի **«Ավելացնել»** կոնտեքստային ֆունկցիայով։

![notification_groups](images/jobs/notification_groups.png)

Սահմանում է այն օգտագործողների խումբը, որոնց առաջադրանքների կատարման ընթացքում ուղարկվելու են ծանուցումներ։ Առաջադրանքի գործողությունների ծանուցումները կարգավորվում են [«Առաջադրանքի ձևանմուշ»](#առաջադրանքի-ձևանմուշ) փաստաթղթի միջոցով, իսկ ամբողջ առաջադրանքի վերաբերյալ ծանուցումները միացվում են **«Համակարգային իրադարձությունների ծանուցումներ»** դիալոգում։

![notification_group](images/jobs/notification_group.png)

| Անվանում | Նկարագրություն |
|---------|----------------|
| Անվանում | Ծանուցման խմբի կոդը (id): |
| Նկարագրություն | Ծանուցման խմբի նկարագրությունը։ |
| Էլեկտրոնային փոստով | Այն օգտագործողների էլ․ փոստերի ցուցակը, որոնք ստանալու են ծանուցում։ Control-ի DropDown-ում ցուցադրվում են **EMAILADDRESS** համակարգային պարամետրում ավելացված էլ․ փոստի հասցեները։ Հնարավոր է նաև ձեռքով մուտքագրել էլ․ փոստերի հասցեները՝ իրարից տարանջատելով ; սիմվոլներով։  |

## Համակարգային իրադարձությունների ծանուցումներ

Այս դիալոգը գտնվում է «Ադմինիստրատորի ԱՇՏ» -> «Համակարգային աշխատանքներ» թղթապանակում։ Այն տալիս է հնարավորություն ընտրելու այն ծանուցման խմբերը, որոնք ծանուցվելու են առաջադրանքի կատարման ընթացքում։ Արդյունքում ընտրված խմբերի կոդերը ավելացվում են **8XJOBNOTIFICATION** համակարգային պարամետրում։
