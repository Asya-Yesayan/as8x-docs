# DocumentChangeRequest դաս

## Ներածություն

Փաստաթղթի պահպանման ընթացքում դրանում կատարված փոփոխությունները կիրառվում և գրանցվում են համակարգում անմիջապես։

Սակայն գոյություն ունեն փաստաթղթեր, որոնք նախքան համակարգում գրանցումը պետք է անցնեն լրացուցիչ ստուգումներ այլ օգտագործողի կամ համակարգի կողմից։ Այսպիսի փաստաթղթերի օրինակներ են՝

- հաճախորդի անձնական տվյալների փոփոխությունները,
- հաշվի կարգավորումների փոփոխությունները,
- քարտի տվյալների փոփոխությունները։

Նման դեպքերում փոփոխությունների պահպանման ընթացքում փաստաթղթի համար ստեղծվում է **Փաստաթղթի փոփոխման հայտ**, որը հետագայում անցնում է հաստատման կամ մերժման փուլ։ Հաստատման դեպքում փոփոխությունները կիրառվում են փաստաթղթի վրա, և այն գրանցվում է համակարգում, մերժման դեպքում փոփոխությունները չեն կիրառվում։

## Փաստաթղթի փոփոխման հայտով գրանցվող փաստաթղթի սահմանումը

Փաստաթղթի փոփոխման հայտով գրանցվող փաստաթուղթ սահմանելու համար անհրաժեշտ է ստեղծել [Document](../Definitions/Document/Document.md)-ից ժառանգվող դաս, որը իրականացնում է `IDocumentChangeRequest` ինտերֆեյսը։

```c#
[Document("Customer",
          ArmenianCaption = "Հաճախորդ",
          EnglishCaption = "Customer")]
public class Customer : Document, IDocumentChangeRequest
{
    public Task ChangeRequest(ChangeRequestEventArgs args) { ... }
}
```

`IDocumentChangeRequest` ինտերֆեյսի [ChangeRequest](IDocumentChangeRequest/Methods/ChangeRequest.md) մեթոդը որոշում է՝ արդյոք փաստաթղթի գրանցման ընթացքում կստեղծվի փոփոխման հայտ, և անհրաժեշտության դեպքում հնարավորություն է տալիս հայտը միանգամից ստեղծել ու հաստատել/մերժել։

## Փաստաթղթի փոփոխման հայտի ստեղծման պայմանները

Փաստաթղթի փոփոխման հայտը ստեղծվում է **գոյություն ունեցող փաստաթղթի խմբագրման ընթացքում**՝ միաժամանակ հետևյալ պայմանների առկայության դեպքում՝

- փաստաթղթի դասը իրականացնում է `IDocumentChangeRequest` ինտերֆեյսը,
- խմբագրվում է արդեն իսկ համակարգում գրանցված փաստաթուղթը (նոր ստեղծվող փաստաթղթի առաջին պահպանման ընթացքում փոփոխման հայտ չի ստեղծվում՝ փաստաթուղթը գրանցվում է անմիջապես՝ առանց հաստատման փուլի),
- [ChangeRequest](IDocumentChangeRequest/Methods/ChangeRequest.md) մեթոդը որոշում է, արդյոք փոփոխությունների համար անհրաժեշտ է ստեղծել հայտ։

Համապատասխանաբար UI-ից փոփոխման հայտ ստեղծելու համար անհրաժեշտ է բացել նախկինում գրանցված փաստաթուղթը խմբագրման համար, կատարել փոփոխությունները և պահպանել. պահպանման արդյունքում փոփոխությունները ոչ թե կիրառվում են փաստաթղթի վրա, այլ գրանցվում են որպես փոփոխման հայտ՝ սպասելով հաստատման կամ մերժման։

## Փաստաթղթի փոփոխման հայտի հաստատումը և մերժումը

### Ծրագրային եղանակով

Փաստաթղթի փոփոխման հայտը ծրագրային եղանակով հաստատելու կամ մերժելու համար նախատեսված են [IDocumentChangeRequestService](../Services/IDocumentChangeRequest/IDocumentChangeRequestService.md) սերվիսի `Confirm` և `Reject` մեթոդները։ `Confirm`-ի կանչի արդյունքում փոփոխման հայտի փոփոխությունները կիրառվում են փաստաթղթի վրա, և այն գրանցվում է համակարգում, `Reject`-ի կանչի արդյունքում հայտը մերժվում է առանց փոփոխությունների կիրառման։

### UI-ից

Ստեղծված Փաստաթղթի փոփոխման հայտերի դիտումը, հաստատումը և մերժումը կարող են կատարվել **«Իրավասությունների փոփոխությունների հաստատում»** ԱՇՏ-ի **«Փաստաթղթերի փոփոխման հայտեր»** դիտելու ձևի միջոցով։

## Հաստատման ընթացքում կատարվող գործողությունները

Փաստաթղթի փոփոխման հայտի հաստատման ընթացքում փաստաթղթի հետ կապված լրացուցիչ գործողությունները հնարավոր է սահմանել երկու եղանակով՝

- [Document](../Definitions/Document/Document.md)-ից ժառանգված դասում [OnConfirmDocumentChangeRequest](../Definitions/Document/Methods/OnConfirmDocumentChangeRequest.md) մեթոդի վերասահմանմամբ (օրինակ՝ ծանուցումների ուղարկում, հաստատման մասին հավելյալ լոգի գրառում, կապակցված տվյալների թարմացում),
- [DocumentExtender](../Extensions/Definitions/DocumentExtender/document_extender.md)-ից ժառանգված դասում [PreOnConfirmDocumentChangeRequest](../Extensions/Definitions/DocumentExtender/Methods/PreOnConfirmDocumentChangeRequest.md) և [PostOnConfirmDocumentChangeRequest](../Extensions/Definitions/DocumentExtender/Methods/PostOnConfirmDocumentChangeRequest.md) մեթոդների վերասահմանմամբ՝ առանց փաստաթղթի դասը փոփոխելու։ Մեթոդները նախատեսված են հաստատման հիմնական մշակմանը համապատասխանաբար նախորդող և հաջորդող տրամաբանության սահմանման համար (օրինակ՝ նախնական ստուգումներ, արտաքին համակարգերում թարմացման մասին հաղորդում)։

## Մերժման ընթացքում կատարվող գործողությունները

Փաստաթղթի փոփոխման հայտի մերժման ընթացքում փաստաթղթի հետ կապված լրացուցիչ գործողությունները հնարավոր է սահմանել երկու եղանակով՝

- [Document](../Definitions/Document/Document.md)-ից ժառանգված դասում [OnRejectDocumentChangeRequest](../Definitions/Document/Methods/OnRejectDocumentChangeRequest.md) մեթոդի վերասահմանմամբ (օրինակ՝ մերժման մասին ծանուցումների ուղարկում, մերժման մասին հավելյալ լոգի գրառում, ընթացիկ փաստաթղթի վիճակի վերականգնում),
- [DocumentExtender](../Extensions/Definitions/DocumentExtender/document_extender.md)-ից ժառանգված դասում [PreOnRejectDocumentChangeRequest](../Extensions/Definitions/DocumentExtender/Methods/PreOnRejectDocumentChangeRequest.md) և [PostOnRejectDocumentChangeRequest](../Extensions/Definitions/DocumentExtender/Methods/PostOnRejectDocumentChangeRequest.md) մեթոդների վերասահմանմամբ՝ առանց փաստաթղթի դասը փոփոխելու։ Մեթոդները նախատեսված են մերժման հիմնական մշակմանը համապատասխանաբար նախորդող և հաջորդող տրամաբանության սահմանման համար (օրինակ՝ նախնական ստուգումներ, արտաքին համակարգերում մերժման մասին հաղորդում)։

{: .important }
Հաստատման և մերժման իրադարձության վերը թվարկված մշակիչները կանչվում են համակարգի կողմից փաստաթղթի փոփոխման հայտի հաստատման կամ մերժման ընթացքում՝ անկախ այն բանից, թե հայտը հաստատվել/մերժվել է ծրագրային եղանակով, թե UI-ից։

| ------------------ | ------------- | --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| SUID               | short         |      -           | Փաստաթղթի փոփոխման հայտը ստեղծած օգտատիրոջ ներքին համարը (id)։ |

| EmptyMessage       | string        |      -           | Դատարկ փաստաթղթի հայտ գրանցել փորձելիս առաջացող սխալի հաղորդագրությունը։ Եթե արժեքը դատարկ տող է, ապա առաջանում է ստանդարտ տեքստով սխալ։ |
| ReFolderDocument   | bool          |      -           | true արժեքի դեպքում փաստաթղթի փոփոխման հայտը մերժելիս փաստաթուղթը [վերաինդեքսավորում է թղթապանակներում](../Services/IDocument/Methods/ReFolder.md)։ |
| CreationLogMessage | string        |       -          | Փաստաթղթի փոփոխման հայտի ստեղծման ժամանակ [փաստաթղթի պատմությունում](https://armsoft.github.io/as4x-docs/HTM/ProgrGuide/Database/DocLog.html) գրանցվող հաղորդագրությունը։ |
