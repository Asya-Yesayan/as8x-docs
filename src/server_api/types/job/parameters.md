## SEQREADURL, SEQAPIKEY

Համակարգում գրանցվող լոգերը ուղարկվում են Seq սերվերին, որի հասցեն սահմանվում է appsettings.json կոնֆիգուրացիոն ֆայլում՝ Serilog:WriteTo:Args:serverUrl դաշտի միջոցով։

**SEQREADURL** պարամետրը նախատեսված է Seq սերվերի հասցե պահպանելու համար։

առաջադրանքի կատարման ընթացքում գրանցված սխալները հնարավոր է տեսնել «Ադմինիստրատորի աշտ» -> «Համակարգային աշխատանքներ» -> «Առաջադրանքներ» թղթապանակի «Առաջադրանքների պատմություն», «Առաջադրանքների պատմություն (ըստ քայլերի)» ձևերում՝ «Կատարման ընթացքում առաջացած սխալներ» կամ «Կատարման ընթացքում առաջացած սխալներ Seq-ում» կոնտեքստային ֆունկցիաների միջոցով։ հնարավոր է տեսնել առաջադրանքի կատարման ընթացքում գրանցված սխալները։

Նշված կոնտեքստային ֆունկցիաները Seq սերվերին միանալու համար նախ ստուգում են SEQREADURL պարամետրի արժեքը։ Եթե այն դատարկ է, օգտագործվում է Serilog:WriteTo:Args:serverUrl կոնֆիգուրացիայի արժեքը, իսկ միացման համար կիրառվում է SEQAPIKEY պարամետրը։

## 8XJOBNOTIFICATION

Այս պարամետրը նախատեսված է [ծանուցման խմբերի](../../../UI/jobs.md#ծանուցման-խումբ) կոդերի սահմանման, որոնք ծանուցվելու են [առաջադրանքների](../../../UI/jobs.md#առաջադրանքի-ձևանմուշ) կատարման ընթացքում։ Խմբերի կոդերը անհրաժեշտ է իրարից տարանջատել ստորակետերով։
